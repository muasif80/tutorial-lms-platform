Scholr LMS — live API console

A lightweight console that drives the running multi-tenant backend (Parts 1–10). Create tenants, create courses, watch tenant isolation hold, enroll learners idempotently, and merge offline progress conflict-free — all against the real REST API.
This is a demo console, not the production frontend. The production UI (Next.js SSR, a design system, PWA + offline, WCAG 2.2 AA) is the subject of Part 9 — this page exists so you can see the backend work in a browser.

1 · Tenants (multi-tenancy)

Each organization is a tenant. Everything below is scoped to the selected tenant via the X-Tenant-Id header.

2 · Courses & tenant isolation

Create a course in the active tenant, then switch tenants — each sees only its own. (Hibernate @TenantId + Postgres RLS.)

3 · Idempotent enrollment

Create a learner + a seat-limited cohort, then enroll twice — you get the same enrollment back, never a duplicate.


  


  
  

    
4 · Offline sync (Part 9)

    
Two devices make progress offline and reconnect. Completed lessons merge by set union (conflict-free); the position cursor resolves by last-write-wins.
